## 括号匹配问题
括号匹配是栈的经典应用场景，在括号匹配过程中，出现无法匹配问题有且仅有以下三种基础情况：
1. 左括号多余
2. 右括号多余
3. 括号没有多余但是对位括号类型不一致

其余情况仅为以上三种基础情况的排列组合.

>Trick：在匹配左括号的时候，右括号先入栈，在遇到右括号时就只需要比较当前元素和栈顶相不相等就可以了，这比左括号先入栈代码实现要简单的多了！

针对以上问题逐个分析代码执行思路：

1. 左括号多余时，字符串已经遍历结束，但是栈不为空（stack.empty == false），return false；
2. 右括号多余时，栈已经为空了（stack.empty == true），但是字符串还未遍历结束，return false；
3. 括号没有多余但是对位括号类型不一致时，当前元素与栈顶元素不相同，return false。
由此得到以下执行代码：

```C++
// 已经由Leetcode验证

bool isValid(string s) {
        if (s.size() % 2 != 0) return false; // 如果s的长度为奇数，一定不符合要求
        stack<char> st;
        for (int i = 0; i < s.size(); i++) {
            // 在匹配左括号的时候，右括号先入栈，就只需要比较当前元素和栈顶相不相等就可以了，比左括号先入栈代码实现要简单的多了！
            if (s[i] == '(') st.push(')');
            else if (s[i] == '{') st.push('}');
            else if (s[i] == '[') st.push(']');
            // 第三种情况：遍历字符串匹配的过程中，栈已经为空了，没有匹配的字符了，说明右括号没有找到对应的左括号 return false
            // 第二种情况：遍历字符串匹配的过程中，发现栈里没有我们要匹配的字符。所以return false
            else if (st.empty() || st.top() != s[i]) return false;
            else st.pop(); // st.top() 与 s[i]相等，栈弹出元素
        }
        // 第一种情况：此时我们已经遍历完了字符串，但是栈不为空，说明有相应的左括号没有右括号来匹配，所以return false，否则就return true
        return st.empty();
    }

```
